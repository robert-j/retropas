# Retro Pascal Code Extension

> These are generic developer *Getting started* docs as generated by `Yeoman`
> and slightly modified to match our project better.

## What's in the folder

* This folder contains all of the files necessary for your extension.
* `package.json` - this is the manifest file in which you declare your
  language support and define the location of the grammar file that has been
  copied into your extension.
* `grammar/*.grammar.json` - these are the TextMate grammar files used
  for tokenization.
* `config/*.config.json` - these are the language configurations, defining
  the tokens that are used for comments and brackets.

## Get up and running straight away

* Press `F5` to open a new window with your extension loaded.
  The launch config uses `--disable-extensions` so only `retropas` is active.
* Load the files in `tests` and verify that syntax highlighting works and
  that the language configuration settings are working.
* **Note:** The extension does not claim file extensions (to avoid overriding
  other users' Pascal/ASM associations). During development, you'll need to
  manually select the language mode ("Turbo Pascal" or "TASM") via the status
  bar — or temporarily add `extensions` back to `package.json`:

  ```json
  { "id": "turbo", "aliases": ["Turbo Pascal"], "extensions": [".pas", ".inc", ".dpr", ".dpk"], ... }
  { "id": "tasm", "aliases": ["TASM"], "extensions": [".asm"], ... }
  ```

## Make changes

* You can relaunch the extension from the debug toolbar after making changes
  to the files listed above.
* You can also reload (`Ctrl + R` or `Cmd + R` on Mac) the VS Code window with
  your extension to load your changes.

## Add more language features

* To add features such as IntelliSense, hovers and validators check out
[Language Extensions Overview](https://code.visualstudio.com/api/language-extensions/overview).

## Install your extension

* To start using your extension with Visual Studio Code,
  * close Code
  * copy this folder (`retropas`) into the `~/.vscode/extensions` folder
  * remove `~/.vscode/extensions/extensions.json`
  * restart Code.
* To share your extension with the world, refer to [Publishing Extensions](https://code.visualstudio.com/api/working-with-extensions/publishing-extension).
